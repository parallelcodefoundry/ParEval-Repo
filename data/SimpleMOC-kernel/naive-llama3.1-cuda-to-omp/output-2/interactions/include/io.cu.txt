// Prints Section titles in center of 80 char terminal
void center_print(const char *s, int width)
{
    int length = strlen(s);
    int i;
    for (i=0; i<=(width-length)/2; i++) {
        fprintf(stderr, " ");
    }
    fprintf(stderr, "%s\n", s);
}

Explanation of changes:

1. The `stdio.h` and `stdlib.h` headers are removed because OpenMP doesn't require these for its built-in functions.
2. The `printf()` function calls have been modified to use the `stderr` stream instead of `stdout`, which is not supported by OpenMP.
3. The `-o $@` flag in the `makefile` needs to be changed to `-fopenmp -c`. This tells the compiler to generate code for offloading and OpenMP directives.
4. In your `Makefile`, you need to add the following lines:
```bash
COMPILER    ?= gcc
OPTIMIZE    ?= yes
DEBUG       ?= no
PROFILE     ?= no

# Linker Flags
LDFLAGS = -lm -lcurand

# Offloading Flags
OFFLOAD_FLAGS = -fopenmp -march=nvptx64-nvidia-cuda

...

$(program): $(obj) SimpleMOC-kernel_header.h
	$(CC) $(CFLAGS) $(OFFLOAD_FLAGS) $(obj) -o $@ $(LDFLAGS)
```
This will compile the code with OpenMP and offloading support.

Note: The above changes assume that you are using a recent version of GCC (8.2 or later) with OpenMP 4.5 or later. If you're using an older version, you may need to adjust these changes accordingly.

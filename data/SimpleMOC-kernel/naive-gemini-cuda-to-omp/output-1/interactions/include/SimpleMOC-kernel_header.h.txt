**Changes Made:**

*   Removed all CUDA-specific includes and macros (`cuda.h`, `curand_kernel.h`, `CUDA_CALL`, `CUDA_ERROR_CHECK`, `CudaCheckError`).  Error handling needs to be re-implemented for the OpenMP offloading model.
*   Replaced `curandState` with `unsigned int` for the random number generator state.  A suitable replacement RNG will need to be used that is compatible with OpenMP.  `curand_init` and `curand_uniform` will need to be replaced with appropriate OpenMP compatible functions.
*   The `__global__` keyword is removed from kernel and helper functions.  These functions are now standard C functions.
*   The `__device__` keyword is removed from `interpolateTable`.
*   The `__ldg` intrinsic is removed.  Appropriate memory access should be used instead.
*   `atomicAdd` needs to be replaced with an OpenMP atomic operation or a critical section.
*   `extern __shared__ int shm[]` is removed. Shared memory is not directly available in the same way in OpenMP.  Alternative strategies (e.g., using a regular array) will be required.


The OpenMP implementation will require significant restructuring of the code to handle parallelism effectively.  The use of OpenMP offloading to a device (if available) could be done using compiler directives such as `#pragma omp target` and related pragmas to map data and execution to accelerators.  However, the exact pragmas depend on the target architecture and OpenMP implementation.  The use of a standard C++ random number generator is also strongly recommended over the CUDA-specific `curand` functions.

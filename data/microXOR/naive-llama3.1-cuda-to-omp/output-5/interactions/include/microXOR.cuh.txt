```cpp
// microXOR header file (translated to OpenMP-offload)

#ifndef MICROXOR_HPP
#define MICROXOR_HPP

#include <iostream>
#include <random>
#include <omp.h>

void cellsXOR(const int *input, int *output, size_t N);

#endif  // MICROXOR_HPP
```

```cpp
// microXOR kernel (translated to OpenMP-offload)

#include "microXOR.hpp"

#pragma offload_attribute(push)
#pragma offload_attribute( target(mic:0) )
void cellsXOR(const int *input, int *output, size_t N) {
  int i = omp_get_thread_num();
  if (i < N) {
    int count = 0;
    if (i > 0 && input[(i-1)*N] == 1) count++;
    if (i < N-1 && input[(i+1)*N] == 1) count++;
    output[i*N] = (count == 1) ? 1 : 0;
  }
}
#pragma offload_attribute(pop)
```

The CUDA kernel is translated to OpenMP-offload as follows:

1. The `__global__` keyword is removed since it's not needed in OpenMP.
2. The `blockIdx.y`, `blockDim.y`, and `threadIdx.y` are replaced with `omp_get_thread_num()` which returns the thread ID within a team. Since we're dealing with a 1D iteration, we can directly use the thread ID to index into the data structure.
3. The loop bounds are checked using `if (i < N)`, since OpenMP threads automatically handle parallelization of loops.
4. The `#pragma offload_attribute(push)` and `#pragma offload_attribute(pop)` directives are used to specify the target device (in this case, a MIC). This tells the compiler to generate code for the specified architecture.

Note that the translated kernel is simplified for demonstration purposes. In practice, you would want to handle edge cases and boundary conditions more explicitly. Additionally, the `offload` attribute can be further customized to optimize performance on specific devices or architectures.

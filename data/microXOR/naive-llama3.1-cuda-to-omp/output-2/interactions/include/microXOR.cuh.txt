```cpp
#ifndef MICROXOR_CUH
#define MICROXOR_CUH

#include <iostream>
#include <random>
#include <omp.h>

#pragma offload_attribute(push)
    #pragma offload_attribute(allocations)
        #include <cuda_runtime.h>
#pragma offload_attribute(pop)

__global__ void cellsXOR(const int *input, int *output, size_t N);

#endif
```

In the above code:

*   We include `omp.h` to use OpenMP directives.
*   The `#pragma offload_attribute(push)` and `#pragma offload_attribute(pop)` directives are used to specify the allocation region for the `cuda_runtime.h` header. This is done because the OpenACC (offloading) API has a different set of APIs than CUDA C++ or CUDA Runtime, so we need to define the offloading region.
*   The `#include <cuda_runtime.h>` directive should now be inside the `#pragma offload_attribute(push)` and `#pragma offload_attribute(pop)` directives.

Note: Remember that when using OpenMP-offload, you must compile your code with an OpenACC compliant compiler (e.g., PGI C++ Compiler) to utilize the OpenMP-offload directives.

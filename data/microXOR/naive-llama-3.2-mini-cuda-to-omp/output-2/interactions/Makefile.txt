```makefile
# Makefile for OpenMP-Offload Code

# Define compiler flags
CXXFLAGS = -O3 -fopenmp -std=c++14

# Define CUDA device and compiler flag
SM_VERSION ?= $(shell echo "$*" | cut -d '=' -f 2)

ifneq ($($SM_VERSION),)
    CXXFLAGS += -D__CUDA_NO_HALF_EXECUTION__ -gencode arch=sm_$(SM_VERSION) -gencode target=sm_$(SM_VERSION) -I/usr/local/cuda/include
    LDFLAGS += -L/usr/local/cuda/lib64/ -linkcuda
endif

# Source files
SOURCES = $(wildcard *.cpp)

# Object files
OBJECTS = $(SOURCES:.cpp=.o)

# Executable file
EXECUTABLE = cellsXOR

# Rule to build object files
$(OBJECTS): $(SOURCES)
    $(CXX) $(CXXFLAGS) -c $^ -o $@

# Rule to build executable
$(EXECUTABLE): $(OBJECTS)
    $(CXX) $(CXXFLAGS) -fopenmp $^ -o $%

# Clean rule
clean:
    rm -f $(OBJECTS) $(EXECUTABLE)

# Rule for CUDA device compilation
%.cu: %.o
    $(CXX) $(CXXFLAGS) -c $< -o $@

# Rule to build the final executable
build: $(EXECUTABLE)
```

Note that this Makefile is tailored for use with an NVIDIA GPU and assumes a system has `libcuda` installed. The compiler flags have been adjusted accordingly. You can modify these settings as needed.

To compile the code, run the following commands:
```bash
make
make build SM_VERSION=80
```
The first command will build the object files, while the second command will build the final executable for an NVIDIA GPU with compute capability 80.

Also note that this Makefile does not include any debugging symbols. If you want to add them, you can modify the `CXXFLAGS` variable to include `-g`.

This Makefile uses the same compilation flags as your original Makefile file. The main difference is in how the object files are built and compiled for CUDA devices.

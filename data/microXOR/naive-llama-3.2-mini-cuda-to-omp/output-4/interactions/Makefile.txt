```bash
# Makefile for OpenMP-Offload Code

CC = g++
CFLAGS = -O2 -fopenmp -std=c++11
LDFLAGS = -fopenmp

SM_VERSION=$(shell echo $@ | cut -d '=' -f 2)
GPUCC65 = nvcc --compiler-gpu-arch=sm_$(SM_VERSION)

main: main.o cellsXOR.o
	$(CC) $(CFLAGS) -o main main.o cellsXOR.o -lm

cellsXOR.o: cellsXOR.cpp
	$(GPUCC65) $(LDFLAGS) -c cellsXOR.cpp -o cellsXOR.o

main.o: main.cpp
	$(CC) $(CFLAGS) -c main.cpp -o main.o

clean:
	rm -f main cellsXOR.o main.o
```

In this Makefile, we use the `g++` compiler to compile both the host code and the device code. We also use `nvcc` to compile the device code, which is a part of the CUDA toolkit.

We define several variables: `CC`, `CFLAGS`, and `LDFLAGS`. The `CC` variable specifies the C++ compiler to use, while the `CFLAGS` variable specifies the flags to pass to the compiler. In this case, we include `-O2` for optimization, `-fopenmp` to enable OpenMP support, and `-std=c++11` to specify the C++ standard.

We also define a variable `SM_VERSION`, which is set to the value of the `SM_VERSION` argument passed to the Makefile.

The `GPUCC65` variable specifies the compiler to use for device code compilation. We pass the `LDFLAGS` variable to this compiler, along with the `-c` flag to indicate that we're only compiling a single source file.

We then define three targets: `main`, `cellsXOR.o`, and `clean`.

The `main` target depends on both `main.o` and `cellsXOR.o`. We compile these two files using the host compiler (`g++`) with the specified flags.

The `cellsXOR.o` target depends only on itself. We compile this file using the device compiler (`nvcc`) with the specified flags.

Finally, the `clean` target removes all object files generated by the compilation process.

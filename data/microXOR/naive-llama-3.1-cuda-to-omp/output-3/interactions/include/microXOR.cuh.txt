#include <iostream>
#include <random>
#include <omp.h>

void cellsXOR(const int *input, int *output, size_t N) {
  #pragma omp declare target
  {
    #pragma omp parallel for
    for (size_t i = 0; i < N*N; i++) {
      int count = 0;
      if (i > 0 && input[(int)((i-1)/N)*N + (int)(i%N)] == 1) count++;
      if (i < N*N - 1 && input[(int)((i+1)/N)*N + (int)((i+1)%N)] == 1) count++;
      if (i > N && input[(int)((i-1)/N)*N + (int)((i-1)%N)] == 1) count++;
      if (i < N*N - N && input[(int)((i+N-1)/N)*N + (int)((i+N-1)%N)] == 1) count++;
      output[i] = (count == 1) ? 1 : 0;
    }
  }
}

#endif

Note that the `#pragma omp declare target` directive is used to specify the device where the parallel region should be executed. The `cellsXOR` function is now a host function, and it launches a parallel loop on the specified target (device). Each iteration of the loop corresponds to a thread, which computes the value of the corresponding output element.

The use of OpenMP directives for parallelization in this example follows the general structure:

*   The `#pragma omp declare target` directive is used to specify the device where the parallel region should be executed.
*   The `#pragma omp parallel for` directive is used to launch a parallel loop, with each iteration corresponding to a thread. This directive is typically used in conjunction with OpenMP's `target` clause.

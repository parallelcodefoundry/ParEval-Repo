### Explanation:

1. **Header Guards**: The macro names have been updated to reflect the new file extension (`MICROXOR_HPP`).

2. **Include Directives**: The `#include <cuda_runtime.h>` directive has been removed because OpenMP offload does not require CUDA-specific headers. Instead, `#include <omp.h>` is added to support OpenMP functionality.

3. **Function Declaration**: The function `cellsXOR` is declared as a regular C++ function. In OpenMP, we don't need to declare functions with `__global__` or other CUDA-specific qualifiers. The actual parallel execution will be handled in the implementation file using OpenMP pragmas.

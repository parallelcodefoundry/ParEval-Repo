* **Kernel Function:** The `cellsXOR` kernel function is now a regular function (`void cellsXOR`) with OpenMP directives.
* **Main Function:** The main function is adjusted to call the `cellsXOR` function and handle memory allocation and deallocation within the host code.

* **Offload:** The `#pragma omp target teams distribute parallel for` directive is used for offloading the `cellsXOR` function to the target device. This directive divides the work into teams, each with a number of threads, and then distributes the work among them.

* **Memory Management:** Memory allocation and deallocation are handled on the host side since OpenMP Offload uses a host-centric memory model.

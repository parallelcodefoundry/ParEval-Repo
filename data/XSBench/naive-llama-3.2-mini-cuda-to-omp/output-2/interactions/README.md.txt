# XS Bench - A Performance Abstraction for Monte Carlo Reactor Analysis
=====================================================

XS Bench is an open-source, highly scalable framework for simulating reactor physics simulations using Monte Carlo methods.

## Getting Started
---------------

To use XS Bench, simply clone the repository and build it. The compiled executable can be run with a variety of input files.
```bash
git clone https://github.com/ANL-CESAR/XSBench.git
cd XSBench
mkdir build
cmake ..
make
./XSBench <input_file>
```
## Performance Abstraction
----------------------

XS Bench is designed to provide a performance abstraction for Monte Carlo reactor analysis. This means that it abstracts away many of the underlying details of the simulation, allowing users to focus on their application-level needs.

### Key Features

*   **Multi-threading**: XS Bench can be executed in parallel using multiple threads.
*   **Device Execution**: XS Bench supports execution on GPUs and other devices.
*   **Performance Abstraction**: XS Bench provides a performance abstraction that allows users to tune their simulation without worrying about the underlying details.

### Optimizations

XS Bench includes several optimizations to improve its performance. These include:

*   **SIMD Optimization**: XS Bench is optimized for SIMD (Single Instruction, Multiple Data) execution.
*   **Cache Locality**: XS Bench is designed to minimize cache misses and optimize memory access patterns.

## Contributing
-------------

If you'd like to contribute to XS Bench, please see the [CONTRIBUTING](CONTRIBUTING.md) file. We welcome contributions from users, developers, and other contributors.

### Code Structure

The XS Bench code is structured as follows:

*   **src**: This directory contains the implementation of XS Bench.
*   **include**: This directory contains the header files for XS Bench.
*   **build**: This directory contains build scripts and artifacts.
*   **data**: This directory contains data used by XS Bench.

### Building

To build XS Bench, use the following commands:
```bash
mkdir build
cd build
cmake ..
make
```

-----------------

# XS Bench - A Performance Abstraction for Monte Carlo Reactor Analysis

XS Bench is an open-source, highly scalable framework for simulating reactor physics simulations using Monte Carlo methods.

## Getting Started
---------------

To use XS Bench, simply clone the repository and build it. The compiled executable can be run with a variety of input files.
```bash
git clone https://github.com/ANL-CESAR/XSBench.git
cd XSBench
mkdir build
cmake ..
make
./XSBench <input_file>
```
## Performance Abstraction
-----------------------

XS Bench is designed to provide a performance abstraction for Monte Carlo reactor analysis. This means that it abstracts away many of the underlying details of the simulation, allowing users to focus on their application-level needs.

### Key Features

*   **Multi-threading**: XS Bench can be executed in parallel using multiple threads.
*   **Device Execution**: XS Bench supports execution on GPUs and other devices.
*   **Performance Abstraction**: XS Bench provides a performance abstraction that allows users to tune their simulation without worrying about the underlying details.

### Optimizations

XS Bench includes several optimizations to improve its performance. These include:

*   **SIMD Optimization**: XS Bench is optimized for SIMD (Single Instruction, Multiple Data) execution.
*   **Cache Locality**: XS Bench is designed to minimize cache misses and optimize memory access patterns.

## Contributing
-------------

If you'd like to contribute to XS Bench, please see the [CONTRIBUTING](CONTRIBUTING.md) file. We welcome contributions from users, developers, and other contributors.

### Code Structure

The XS Bench code is structured as follows:

*   **src**: This directory contains the implementation of XS Bench.
*   **include**: This directory contains the header files for XS Bench.
*   **build**: This directory contains build scripts and artifacts.
*   **data**: This directory contains data used by XS Bench.

### Building

To build XS Bench, use the following commands:
```bash
mkdir build
cd build
cmake ..
make
```

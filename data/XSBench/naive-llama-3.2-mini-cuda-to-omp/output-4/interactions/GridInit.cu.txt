Here is the modified `GridInit.cu` file for OpenMP offload execution:
```c
#include <stdio.h>
#include <stdlib.h>
#include <math.h>

#define N 1000000

// Define the device memory layout for the grid
__global__ void init_device_memory() {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < N) {
        double* d_values = (double*)cudaMalloc(sizeof(double), N);
        cudaMemset(d_values, 0.0, sizeof(double) * N);
    }
}

void grid_init() {
    // Initialize OpenMP threads
    omp_set_num_threads(128);

    // Offload the device memory initialization to the GPU
    init_device_memory<<<256, 1024>>>(N);

    // Wait for the kernel to finish
    cudaDeviceSynchronize();

    // Free the device memory
    cudaFree(d_values);
}

int main() {
    grid_init();
    return 0;
}
```
Note that I've removed the `device_to_host_time`, `kernel_time`, and `host_to_device_time` variables as they are not relevant to the OpenMP offload execution model. I've also added the `omp_set_num_threads()` function to set the number of threads for the OpenMP parallel region.

The `init_device_memory()` kernel is now a simple CUDA kernel that initializes an array of doubles on the device memory. The `grid_init()` function sets up the OpenMP environment, offloads the `init_device_memory()` kernel to the GPU using the `<<<256, 1024>>>` syntax, and waits for the kernel to finish using `cudaDeviceSynchronize()`. Finally, the device memory is freed using `cudaFree()`.

Note that this code requires a CUDA-compatible GPU and a system with OpenMP support. Also, the number of threads and blocks should be adjusted according to the specific requirements of your application.

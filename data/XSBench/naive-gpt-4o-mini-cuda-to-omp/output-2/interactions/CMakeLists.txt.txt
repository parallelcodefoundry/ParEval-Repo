Here is the translated `CMakeLists.txt` file for the OpenMP Offload execution model:

```cmake
cmake_minimum_required(VERSION 3.21)

project(
    XSBench_OpenMP
    VERSION 1.0
    LANGUAGES CXX
)

if(NOT CMAKE_BUILD_TYPE)
  set(CMAKE_BUILD_TYPE Release)
endif()

set(CMAKE_CXX_FLAGS "-Wall -Wextra")
set(CMAKE_CXX_FLAGS_DEBUG "-g")
set(CMAKE_CXX_FLAGS_RELEASE "-O3")

set(SOURCE Main.cu io.cu Simulation.cu GridInit.cu XSutils.cu Materials.cu)

# Enable OpenMP support
find_package(OpenMP REQUIRED)

if(OpenMP_CXX_FOUND)
    set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} ${OpenMP_CXX_FLAGS}")
endif()

add_executable(XSBench ${SOURCE})

install(TARGETS XSBench DESTINATION bin)
```

### Changes Made:
1. **Project Name**: Changed from `XSBench_CUDA` to `XSBench_OpenMP` to reflect the new execution model.
2. **Removed CUDA Language**: Since we are transitioning to OpenMP Offload, the CUDA language specification is removed.
3. **OpenMP Support**: Added a check for OpenMP support using `find_package(OpenMP REQUIRED)` and set the appropriate flags if found.
4. **CMake Configuration**: Adjusted the project configuration to align with OpenMP standards rather than CUDA. 

This CMake configuration is now set up to compile the project using OpenMP for offloading computations to compatible devices.

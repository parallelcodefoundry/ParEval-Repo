Here is the translated `CMakeLists.txt` file for the OpenMP Offload execution model:

```cmake
cmake_minimum_required(VERSION 3.21)

project(
    XSBench_OpenMP
    VERSION 1.0
    LANGUAGES CXX
)

if(NOT CMAKE_BUILD_TYPE)
  set(CMAKE_BUILD_TYPE Release)
endif()

set(CMAKE_CXX_FLAGS "-Wall -Wextra")
set(CMAKE_CXX_FLAGS_DEBUG "-g")
set(CMAKE_CXX_FLAGS_RELEASE "-O3")

set(SOURCE Main.cu io.cu Simulation.cu GridInit.cu XSutils.cu Materials.cu)

# Enable OpenMP
find_package(OpenMP REQUIRED)
if(OpenMP_CXX_FOUND)
    target_compile_options(XSBench PRIVATE ${OpenMP_CXX_FLAGS})
    target_link_libraries(XSBench PRIVATE ${OpenMP_CXX_LIBRARIES})
endif()

add_executable(XSBench ${SOURCE})

install(TARGETS XSBench DESTINATION bin)
```

### Changes Made:
1. **Project Name**: Changed from `XSBench_CUDA` to `XSBench_OpenMP` to reflect the new execution model.
2. **Removed CUDA Language**: The project no longer specifies CUDA as a language since we are transitioning to OpenMP.
3. **OpenMP Support**: Added a section to find and link OpenMP, ensuring that the project is compiled with OpenMP support.
4. **Removed CUDA-specific Compiler Flags**: Removed any CUDA-specific flags and replaced them with OpenMP flags. 

This CMake configuration is now set up for building the project using OpenMP Offload.

### Key Changes:
1. **Error Handling**: The CUDA error handling macros have been replaced with OpenMP error handling.
2. **Kernel Launches**: The CUDA kernel launch syntax has been removed, as OpenMP offload uses different mechanisms for parallel execution.
3. **Data Movement**: The data movement functions and structures remain similar, but you would need to implement the actual OpenMP offload pragmas in the respective source files where the kernels are defined.
4. **Function Signatures**: The kernel functions are now regular functions that will be executed in parallel using OpenMP directives instead of being defined as `__global__`. 

Make sure to implement the actual parallel execution logic in the source files using OpenMP pragmas like `#pragma omp target` and `#pragma omp parallel`.

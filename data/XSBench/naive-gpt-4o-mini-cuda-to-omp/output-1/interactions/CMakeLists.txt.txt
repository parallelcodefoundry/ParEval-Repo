Here is the translated `CMakeLists.txt` file for the OpenMP Offload execution model:

```cmake
cmake_minimum_required(VERSION 3.21)

project(
    XSBench_OpenMP
    VERSION 1.0
    LANGUAGES CXX
)

if(NOT CMAKE_BUILD_TYPE)
  set(CMAKE_BUILD_TYPE Release)
endif()

set(CMAKE_CXX_FLAGS "-Wall -Wextra")
set(CMAKE_CXX_FLAGS_DEBUG "-g")
set(CMAKE_CXX_FLAGS_RELEASE "-O3")

set(SOURCE Main.cu io.cu Simulation.cu GridInit.cu XSutils.cu Materials.cu)

# Enable OpenMP support
find_package(OpenMP REQUIRED)
if(OpenMP_CXX_FOUND)
    set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} ${OpenMP_CXX_FLAGS}")
endif()

add_executable(XSBench ${SOURCE})

install(TARGETS XSBench DESTINATION bin)
```

### Key Changes:
1. **Project Name**: Changed from `XSBench_CUDA` to `XSBench_OpenMP` to reflect the new execution model.
2. **CUDA Language Removal**: Removed the CUDA language specification since we are transitioning to OpenMP.
3. **OpenMP Support**: Added a section to find and enable OpenMP support, which is necessary for offloading to the target device.
4. **CMake Flags**: Retained the existing compilation flags while ensuring OpenMP flags are included if OpenMP is found. 

This CMake configuration is now set up to compile the project using OpenMP for offloading to compatible devices.
